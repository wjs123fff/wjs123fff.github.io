随机搜索算法解决TSP问题

一问题引入:

TSP(Traveling Salesman Problem)即旅行商问题，是数学领域中著名问题之一。这个问题是这样的：假设有一个旅行商人要拜访n个城市，他必须选择所要走的路径，路径的限制是每个城市只能拜访一次，而且最后要回到原来出发的城市。路径的选择目标是要求得的路径长度为所有路径之中的最小值。

二问题分析:

假设将上面的问题简化为上面这张图片所描述的内容，那么到底该
如何选择路径，让售货员便利整个城市回到原点的路径和最短呢？
由上图可知，选择不同，路径不同，距离就不同。

三编程思路:

解决这类问题有多种不同的解法，例如模拟退伙法，蚁群迁移法，遗传法，暴力枚举法，贪心法，动态粒子追踪法等等。每种算法都有自己的优缺点以及时间复杂度各有不同。可以根据不同的条件选择合适的算法。
我采用的是基于暴力算法的随机搜索算法，而暴力算法是例举出所有可能的路径，再从其中找到最短的那一条。
例如：
A B C

随机选择就有：ABC, ACB,BAC,BCA,CBA,CAB，6种路径的选择。
而当城市数目过于庞大的时候。这种算法的时间复杂度就不太适合。
所以，随机搜索算法并没有像暴力法那样找出所有可能路径，
而是随机生成N个访问路径，从这随机生成的N条访问路径中挑出最短的那条作为结果返回。
这次实验是基于p5.js的JavaScript库的条件下进行的编码。
第一步：
首先我们要确定城市的数量，
那么，城市坐标呢？所以需要定义一个空数组存储每个城市的点坐标。
我们计算出每次随机产生的路径中的距离，并找到其中最短的一条，所以需要定义一个变量存储当前最短路径。
最后我们需要定义一数组把这条最优路径进行存储。
最终，我们的到了如下的变量：
城市数量：totalCities
存储城市坐标：cities []
当前最短距离总长度：recordDistance
最短路径： bestEver;
如何解决城市随机产生的点坐标呢？
我们JavaScript中学过产生随机数函数，我们只要在一定范围内，将X，Y的点坐标用随机函数进行处理，再储存在cities这个数组内，就可以解决了。


我们随机一个路线出来
0-3-1-2-0（最终是要返回0的）



接下来：随机了一个路径之后，我们要在这个路径下面找到最短的路径，方法是通过交换两个位置得到的，交换任意两个数，如果交换完了之后距离变小了，就更新recordDistance，我们采用的是在当前路径里面随机挑选2个城市，即随机生成两个城市的下标，交换这两个城市在cities里面的位置。

然后，我们就要开始计算每条随机产生的路径的距离，通过比较，的出最短路径最后将路径存入bestEver

我们需要定义一个函数计算两个城市的点坐标并返回:

Dist函数就是欧氏距离算法，计算两个点坐标的距离。定义一个变量d存储计算的距离，如果当前距离小于recordDistance存储的距离，就更新，并将这条路径存入bestEver数组中。
最后用p5.js进行可视化。

我们要明白的是，只要随机的量足够大，最终一定能得到结果，因为能随机到枚举的全部。
是不是感觉最终得不到最优解，可是当你循环个50000次试试，你就知道，基本上99%都能拿到最优解，这个比暴力搜索快，但是比dp不稳，这个算法的有点在于编码不复杂，而且在基本问题上面都能得到最优解，所以最终我选择了这个

总结和展望：

这一次的旅行商问题，我在百度上查找了很多的方法，也看了很多算法的实现原理。我就找了一种比较简单的算法。这个算法也是追求局部最优解而忽视了全局最优的选择。所以不太稳定。但是效率却很高，时间复杂度比较小。这个算法的难点在于随机路径的参数，以及如何在这个路径下找到最短路径。可以通过交换两个城市来进行比较。
总之这一次课题让我有了很大的感触，算法真的特别能锻炼人的思维，同时自己也在不断的思考慢慢的得到了成长。

代码：

var cities = [];
var totalCities = 6;
var recordDistance;
var bestEver;

function setup() {
createCanvas(600,400);
for (var i = 0; i < totalCities; i++) {
var v = createVector(random(width), random(height));
cities[i] = v;
}
var d = calcDistance(cities);
recordDistance = d;
bestEver = cities.slice();
}

function draw() {
background(0);

stroke(255);
strokeWeight(1);
noFill();
beginShape();
for (var i = 0; i < cities.length; i++) {
vertex(cities[i].x, cities[i].y);
}
endShape();

stroke(255, 0, 255);
strokeWeight(4);
noFill();
beginShape();
for (var i = 0; i < cities.length; i++) {
vertex(bestEver[i].x, bestEver[i].y);
}
endShape();

for (let k = 0;k < 20; k++){

var i = floor(random(cities.length));
var j = floor(random(cities.length));
swap(cities, i, j);

var d = calcDistance(cities);
if (d < recordDistance) {
recordDistance = d;
bestEver = cities.slice();
}
}
}
function swap(a, i, j) {
var temp = a[i];
a[i] = a[j];
a[j] = temp;
}
function calcDistance(points) {
var sum = 0;
for (var i = 0; i < points.length - 1; i++) {
var d = dist(points[i].x, points[i].y, points[i + 1].x, points[i + 1].y);
sum += d;
}
return sum;
}

酒店管理系统

酒店管理系统

用例文本：
1.系统功能设计（模块结构图统领下，对各个模块功能进行详尽的文字描述）
应用环境:根据用户的需求陈述可以确定系统分为前台和后台，
前台的主要功能是接待、预订、收银等;后台的主要是管理、统计等。本 系统只能在非网络模式下运行，无论是服务人员、管理人员还是经理， 都要通过系统登录本系统，服务人员通过用户界面获取信息、登记客户 信息;管理人员通过权限对系统进行管理，了解酒店动态。

1.1登录系统:主要执行数据库的运行权限管理，只有用户登录通过才可以进入主页面进行操作。
1.2顾客信息登记系统（订单管理）：主要是记录顾客基本入住信息，包括身份证，姓名，年龄，家庭住址，以及一些入住信息，这个将关系到后面得结算系统。后台管理人员也可以通过查看客源来调整相应的住房情况。

1.3客房标准管理系统:包括客房的编号、类型、面积、最多容纳的人数、
价格以及客房的硬件设施，如果空调、电视、网络、卫生间等。当添加 新的客房时，就添加相应的标准信息，可以随时查看客房的标准信息，
根据需要进行修改和删除等操作。

1.4客房信息管理系统:酒店客房管理系统的核心部分。它的主要功能是 存储订房时的信息，包括登记人姓名、证件号、登记类型、所选的餐饮 标准、押金及入住时间等。这些信息直接关系到后续的结算信息管理。

1.5结算信息管理系统:结合订房的信息进行自动化结账，并存储结算信 息，包括客房定价、押金、餐饮消费、客房消费、客户类型、结算时间、 入住天数、消费总额等, 结算完毕就可以退房。

1． 参与者：

参与者 角色描述
前台人员 接待、订房，收银等
顾客 预定管理
后台管理人员 管理、统计等
（备注：本系统只能在非网络模式下运行，所以所谓的预定管理指的是有一些不是当天入住的顾客）

扒一扒搭建博客踩过的坑

B站是个好地方

某天正在B站”白嫖“计算机网络的时候，偶然间看见一个up主发布了一篇关于如何免费搭建个人博客而且不需要自己掏钱买域名服务器的方法，作为”资深白白嫖客“的我，如何能错过。

1:下载node.js，安装淘宝镜像。
npm install -g cnpm –registry=http://registry.npm.taobao.org #安装淘宝的cnpm 管理器
2：安装hexo博客框架
cnpm install -g hexo-cli #安装hexo框架
hexo -v #查看hexo版本
cd blog #创建blog目录
cd blog #进入blog目录(注：如果电脑是Mac的，命令行是：mkdir)
hexo init #生成博客 初始化博客（mac:mkdir blog #创建blog目录）
hexo s #启动本地博客服务
http://localhost:4000/ #本地访问地址
hexo n “我的第一篇文章” #创建新的文章
#返回blog目录
hexo clean #清理
hexo g #生成
#Github创建一个新的仓库 你的GitHub的用户名.github.io
cnpm install –save hexo-deployer-git #在blog目录下安装git部署插件
（git是为了将项目部署到远端GitHub，对本地的项目管理以及远端项目克隆也是极好的）
#配置_config.yml 

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
      type: git
     repo: https://github.com/YourGithubName/YourGithubName.github.io.git
      branch: master

hexo d #部署到Github仓库里
https://你的GitHub的用户名.github.io/ #访问这个地址可以查看博客

关于更换主题

git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia #下载yilia主题到本地
（注：也可以自己去网上找一款自己喜欢的主题，我用的是black_bluez这一款）

#修改hexo根目录下的 _config.yml 文件 ： theme: yilia

hexo c #清理一下
hexo g #生成
hexo d #部署到远程Github仓库
https://你的GitHub的用户名.io/ #查看博客
