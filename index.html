HTML的 Doctype 作用 ? 严格模式与混杂模式如何区分？它们有何意义 ?

1.< Doctype> 声明位于文档中的最前面，处于html标签之前。告知浏览器以何种模式来渲染文档。

2.所谓的严格模式也称标准模式是指，浏览器按W3C标准解析执行代码；混杂模式也称怪异模式则是使用浏览器自己的方式解析执行代码，因为不同浏览器解析执行的方式不一样，所以我们称之为怪异模式。

3.浏览器解析时到底使用标准模式还是怪异模式，与你网页中的DTD声明直接相关，DTD声明定义了标准文档的类型（标准模式解析）文档类型，会使浏览器使用相应的方式加载网页并显示，忽略DTD声明,将使网页进入怪异模式。DOCTYPE 不存在或形式不正确会导致文档以怪异模式呈现。HTML5 没有 DTD ，因此也就没有严格模式与混杂模式的区别，HTML5 有相对宽松的语法，实现时，已经尽可能大的实现了向后兼容。

package test3;
import java.util.Scanner;
public class Calculate {
private double a;
private double b;

public Calculate(double x, double y) {
    a = x;
    b = y;
}

public double Add() {
    return a + b;
}

public double Minus() {
    return a - b;
}

public double Multiply() {
    return a * b;
}

public double Divide() {
    if(b != 0)
        return a / b;
    else
        System.out.println("除数不能为0！");
    return 0;
}

public static void main(String[] args) {
    Scanner scanner = null;
    scanner = new Scanner(System.in);
    int a = scanner.nextInt();

    int b = scanner.nextInt();

    System.out.println("请输入数字a和b：");

    Calculate Cal = new Calculate(a,b);

    System.out.println("运算结果是：");

    System.out.println("加法"+":"+ a + "+" + b + "="+ Cal.Add());

    System.out.println("减法"+":"+ a + "+" + b +  "="+ Cal.Minus());

    System.out.println("乘法"+":" + a + "+" + b +  "="+ Cal.Multiply());

    System.out.println("除法"+":" + a + "+" + b + "="+ Cal.Divide());
}

}

webpack的打包：

webpack的打包原理我个人理解是，从以往的角度出发，以前的开发在引用js代码时，都是在.html页面中进行引用，但是不同的js文件中也会互相引用,这种引用方式不符合模块化开发的思想，并且浏览器并不识别模块化的代码。所以webpack是一个基于专门打包所以JavaScript代码的一个静态模式的打包器。只要把所以的js文件都引用到同一个js文件上，就可以了，至于其他js文件之间的数据传递都会由webpack进行打包管理。

组件中的data为什么是一个函数

首先，我们来看一下js中的函数


可以看到，当改变obj1对象的属性时，obj2,obj3的属性值并没有发生改变。因为每次调用的时候函数的栈空间每次都会返回一个新对象。
如果将函数改成返回一个obj对象的时候，值会不会跟着改变呢？

因为它们指向的都是同一个内存地址，所有当obj1的值发生改变时，obj2,obj3的值也会发生改变。


同理：当我们组件内创建多个组件实例时，每个组件指向的都是同一个data，相当于资源共享。

这样就有点麻烦了，因为当其中一个实例改变数据时，另外共享同一个data的组件实例也会发生改变




这样应该就不难理解为什么要把数据放在函数里并返回了吧
