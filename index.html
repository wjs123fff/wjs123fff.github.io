扒一扒搭建博客踩过的坑

B站是个好地方

某天正在B站”白嫖“计算机网络的时候，偶然间看见一个up主发布了一篇关于如何免费搭建个人博客而且不需要自己掏钱买域名服务器的方法，作为”资深白白嫖客“的我，如何能错过。

1:下载node.js，安装淘宝镜像。
npm install -g cnpm –registry=http://registry.npm.taobao.org #安装淘宝的cnpm 管理器
2：安装hexo博客框架
cnpm install -g hexo-cli #安装hexo框架
hexo -v #查看hexo版本
cd blog #创建blog目录
cd blog #进入blog目录(注：如果电脑是Mac的，命令行是：mkdir)
hexo init #生成博客 初始化博客（mac:mkdir blog #创建blog目录）
hexo s #启动本地博客服务
http://localhost:4000/ #本地访问地址
hexo n “我的第一篇文章” #创建新的文章
#返回blog目录
hexo clean #清理
hexo g #生成
#Github创建一个新的仓库 你的GitHub的用户名.github.io
cnpm install –save hexo-deployer-git #在blog目录下安装git部署插件
（git是为了将项目部署到远端GitHub，对本地的项目管理以及远端项目克隆也是极好的）
#配置_config.yml 

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
      type: git
     repo: https://github.com/YourGithubName/YourGithubName.github.io.git
      branch: master

hexo d #部署到Github仓库里
https://你的GitHub的用户名.github.io/ #访问这个地址可以查看博客

关于更换主题

git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia #下载yilia主题到本地
（注：也可以自己去网上找一款自己喜欢的主题，我用的是black_bluez这一款）

#修改hexo根目录下的 _config.yml 文件 ： theme: yilia

hexo c #清理一下
hexo g #生成
hexo d #部署到远程Github仓库
https://你的GitHub的用户名.io/ #查看博客

是不是很简单呀？

but， 当时我搭建的时候真的气到爆炸，每走一步，就出现一个错误，特别是下载git那里，我发现在cmd里面就是下载不了，最后不得不用软件管家下载。
若你未关联Github，则执行hexo deploy
命令时终端会提示你输入Github的用户名和密码，即
Username for ‘https://github.com':
Password for ‘https://github.com':

这样呢，是不是就会好了呢，nonono

你的博客不可能都是文字描述吧，古人还说图文并茂呢，所以呀，我迎来了我的又一个大坑，去chrome上访问了很多大佬的博客，然后乐呵呵的去安装步骤一步步进行，安装了上传图片的插件:
1.首先把blog（hexo）目录下的_config.yml里的psot_asset_folder:设置为true

2.在blog（hexo）目录下执行:

cnpm install hexo-asset-image –save

3.在blog（hexo）目录下Git Bash Here，运行hexo n “博客名”来生成md博客时，会在_post目录下看到一个与博客同名的文件夹。

关键是这可是安装大佬的方法一步步进行的哎，咋回事啊，小老弟？
后来找了很多关于这个的博客，差不多是这个方法啊，难不成我的电脑有毒吗？

后来，终于在一个冷门的论坛里面找到了原因：可能是版本原因
cnpm install hexo-asset-image –save
使用这条语句，安装的是0.0.1版本的插件
cnpm install https://github.com/CodeFalling/hexo-asset-image – save
使用这条语句，安装的是0.0.5版本的插件

由于地址映射的不同，0.0.1这个版本的不支持本地图片的映射。
