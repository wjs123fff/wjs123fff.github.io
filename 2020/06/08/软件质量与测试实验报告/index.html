软件质量与测试实验报告

一、题目

1.做程序测试，找出其中的错误，并修正（物品的价格和名称-实验报告中的第一道题，纠错题）

二、问题分析

1.需要新建一个Java项目test，然后将需要测试的product类导入到项目中，再导入JUNIT4测试包。

三、问题实现代码

package test;

public class product {
private String title;
private double price;

public product(String title, double price) {
    this.title = title;
    this.price = price;
}

public String getTitle() {
    return title;
}

public double getPrice() {
    return price;
}

public boolean equals(Object o) {
    if (o instanceof product) {
        product p = (product)o;
        return p.getTitle().equals(title);
    }
    return false;
}

}

插入的数据不应该是0.001而应该是0.01.因为junit中没有assertEquals(double,double)的方法。因为double值是允许误差的。所以要实现double的断言要用assertEquals(double,double,double)这个方法。第三个参数是允许误差 。
测试类中：32.85和32.86误差值应该是0.01.

应该增加调用getTitle()方法，并且定义的“BK”与“book”不一样也要修改。

一、题目

2、设计一个包含加、减、乘、除四则运算的计算器基本功能，使用断言assertEquals对结果的精度进行判断。

二、问题分析

建test2项目，导入JUNIT4测试包。
新定义一个Calculate定义Add()，Minus(),Multiply(),Divide()方法，分别进行加减乘除的运算。
导入测试类，开始进行检测：

三、问题实现代码

package test3;
import java.util.Scanner;
public class Calculate {
private double a;
private double b;

public Calculate(double x, double y) {
    a = x;
    b = y;
}

public double Add() {
    return a + b;
}

public double Minus() {
    return a - b;
}

public double Multiply() {
    return a * b;
}

public double Divide() {
    if(b != 0)
        return a / b;
    else
        System.out.println("除数不能为0！");
    return 0;
}

public static void main(String[] args) {
    Scanner scanner = null;
    scanner = new Scanner(System.in);
    int a = scanner.nextInt();

    int b = scanner.nextInt();

    System.out.println("请输入数字a和b：");

    Calculate Cal = new Calculate(a,b);

    System.out.println("运算结果是：");

    System.out.println("加法"+":"+ a + "+" + b + "="+ Cal.Add());

    System.out.println("减法"+":"+ a + "+" + b +  "="+ Cal.Minus());

    System.out.println("乘法"+":" + a + "+" + b +  "="+ Cal.Multiply());

    System.out.println("除法"+":" + a + "+" + b + "="+ Cal.Divide());
}

}

四、测试代码

package test3;

import static org.junit.Assert.*;

import org.junit.Test;

import junit.framework.TestCase;

public class CalculateTest extends TestCase {
private Calculate a;
private Calculate b;
private Calculate c;
private Calculate d;

public void setUp() throws Exception {
    a = new Calculate (1,99);
    b = new Calculate (100,50);
    c = new Calculate(1,2);
    d = new Calculate(10,5);
}

public void testAdd() {
    assertEquals(100.0,a.Add());
}

public void testMinus() {
    assertEquals(50.0,b.Minus());
}

public void testMultiply() {
    assertEquals(2.0,c.Multiply());
}

public void testDivide() {
    assertEquals(2.0,d.Divide());
}

}

六、总结

第一题总结：
在运行的时候，刚开始一直以为是product类里面的代码出现了错误，所以在那部分代码花了挺久的时间，改了很多部分，后来仔细看了错误，以及参数列表才发现是测试类写的不够完整assertEquals方法调用了解的不够。不过，还是学到了很多。
第二题总结：
我感觉用Java打计数器那部分还是挺简单的，但是到写测试类的地方就有点手足无措了，第一次接触这样的程序形式，但是想到的第一个想法是利用继承关系调用Calculate的构造函数。但是发现不行。第一次失败。
后来我又定义了四个test方法调用assertEquals函数，结果程序不能运行，弹出“Assert 的方法 assertEquals(double, double) 已过时”的错误，有点懵。后来去官网上查了关于assertEquals函数的特点，后来引入了import junit.framework.TestCase;就好了。也能运行和输出一些信息。
这一次学到了很多，也更加了解测试其实在一个软件里面也是很重要的。掌握一些测试方法对开发是很有帮助。
