Vue知识点1

Vue–1day

1.关于let(es6中定义变量)和const(es6定义常量)
let定义得变量的值是可以进行改变的，但是const定义的变量是不能改变的。
var没有作用域
es6中，var的作用域是函数级，let的作用域是块级。
2.vue的基本格式：

 div id="app"{{message}}}/div {{}}里面嵌套的是程序段或数据
const app = new Vue({
        e:'#app',         --->用于挂载要管理的元素
        data:{            --->定义数据
       message:'hello world!'
   }
})

3.vue的数据是动态双向绑定的，与JavaScript和jQuery相比，当数据刷新时，前端加载出来的数据也同步更新，不用再重新编写语句将新数据渲染到页面标签中。（VUE的核心：数据驱动视图。），适用于任何前后端分离的项目。

4.Mustache语法
Mustache语法中，不仅仅可以直接写变量，也可以写简单的表达式

VUE中的MVVM（Model View ViewModel）开发模式:

在MVVM的框架下视图和模型是不能直接通信的。它们通过ViewModel来通信，ViewModel通常要实现一个observer观察者，当数据发生变化，ViewModel能够监听到数据的这种变化，然后通知到对应的视图做自动更新，而当用户操作视图，ViewModel也能监听到视图的变化，然后通知数据做改动，这实际上就实现了数据的双向绑定。并且MVVM中的View 和 ViewModel可以互相通信。

ViewModel的作用：1界面触发事件，通过viewModel回调到model。
2.model更新的数据，通过viewmodel向页面传递。

创建Vue实例传入options

vue的生命周期

什么是生命周期：事物的产生到消亡的过程
关于版本选择：debug(开发版本)/relese（稳定版本）
个人理解：当创建一个实例后，在做每件事的过程中，生命周期会进行回调，告诉程序进行到了哪一步。

生命周期的作用：生命周期给了很多回调，所有要根据项目需求决定要用哪些回调。通过callhook函数传递的数据，hook(钩子)函数来进行回调。其实有点像Java通过监听器来判断实例是否触发，再调用相应接口实现相应的功能。
